# Rules for developers

## 1. Check flutter version before starting.

[flutter --version] make sure it is flutter "3.41.7"

## 2. when you start a new task, create a new branch for it.

* for example to 'New feature' [feature/new-feature] this is the format for branch name.

* if you want to create a branch for bug fix [bugfix/bug-fix-name]

* if you want to create a branch for hot fix [hotfix/hot-fix-name]

* if you want adding thing in [pubspec.yaml] or [pubspec.lock] create a new branch for it [chore/pubspec-update] (get permission from team leader first)

* if you want editing or add thing in design system create a new branch for it [design/design-update] (get permission from team leader first)

## 3. don't use any external packages without permission(from team leader).

* don't use any external packages without permission(from team leader).
* not allowed to use code generation packages like freezed or any other.

## 4. check  flutter doctor before starting.

* [flutter doctor] make sure it is flutter "3.41.7"

## 5. and used  'antigravity' for coding.

* check antigravity rules before starting.
* order .rules file from team leader.
